## 1. 문서 목적

이 문서는 `docs/nodekit-bug-investigation-report-ko.md`에서 확인된 버그들을 실제 수정 스프린트 단위로 정리한 실행 문서다.

이번 스프린트는 단순 핫픽스 묶음이 아니라, **L1 검증 계약을 문서, 테스트, 구현에서 다시 일치시키는 정리 스프린트**로 정의한다.

## 3. 작업 원칙

* 각 버그는 **재현 테스트 또는 실패 케이스 명문화**를 먼저 추가한 뒤 수정한다.
* 문서 기준과 구현 기준이 어긋난 항목은, 구현 수정과 함께 테스트 이름과 설명도 맞춘다.
* UI 상태 버그는 수동 확인만으로 끝내지 않고 가능하면 단위 테스트 또는 최소 회귀 시나리오로 고정한다.
* 휴대성 문제는 “현재 환경에서 동작함”과 “독립 빌드 가능함”을 구분해서 다룬다.

## 5. Phase A — 검증-전송 불일치 해소

**기간**: 2026-04-08(수) ~ 2026-04-10(금) / 3일

**대상 버그**: 4, 5, 1

**달성 목표**: 사용자가 검증한 값과 실제 전송되는 값이 어긋나지 않도록 만들고, 이미지 URI 검증의 명백한 오판을 먼저 차단한다.

| 날짜 | 작업 | 완료 기준 |
|------|------|-----------|
| 04-08 (수) | 스프린트 착수, 버그 4/5/1 재현 케이스 정리 | 재현 시나리오와 대상 테스트 목록 문서화 |
| 04-08 (수) | 버그 4 테스트 추가 | 검증 후 입력 수정 시 전송 금지 또는 재검증 요구 케이스가 테스트로 표현됨 |
| 04-09 (목) | 버그 5 테스트 추가 | `EnvironmentSpec` 보존 여부가 테스트로 고정됨 |
| 04-09 (목) | 버그 1 테스트 추가 | 포트만 있고 태그 없는 이미지, 포트+태그 동시 존재 케이스가 테스트로 고정됨 |
| 04-10 (금) | 버그 4 수정 | 폼 변경 시 `_l1Passed`와 전송 버튼 상태가 무효화됨 |
| 04-10 (금) | 버그 5 수정 | `BuildRequest`에 `EnvironmentSpec`이 포함되고 factory 매핑이 반영됨 |
| 04-10 (금) | 버그 1 수정 | 포트와 태그를 구분하는 이미지 URI 검증이 통과/차단 기준대로 동작함 |

### 리스크 및 대응

| 리스크 | 가능성 | 영향 | 대응 |
|--------|--------|------|------|
| UI 상태 변경 감지가 코드비하인드 전반에 흩어질 가능성 | 중 | 수정 범위 확대 | 입력 변경 공통 무효화 지점을 먼저 설계 |
| `BuildRequest` 변경이 gRPC DTO와 충돌할 가능성 | 중 | 연동 코드 수정 확대 | 수동 DTO와 proto/generated 경계를 먼저 확인 |
| 이미지 URI 규칙이 테스트보다 느슨하게 해석될 가능성 | 중 | 회귀 발생 | 포트/태그/digest 조합 케이스를 최소 4종 고정 |

## 9. 완료 조건

* 버그 1, 4, 5는 반드시 자동 테스트로 회귀 방지 상태가 된다.
* 버그 6, 7, 8은 최소한 문서의 L1 요구사항과 테스트 설명이 어긋나지 않는다.
* 버그 2는 주소 변경 후 실제 새 클라이언트가 사용된다는 근거가 남는다.
* 버그 3은 “현재 환경에서만 동작”과 “독립 빌드 가능”의 차이가 문서에 명확히 기록된다.
* 스프린트 종료 시 미해결 항목은 “잔여 리스크”로 별도 기록한다.

### 현재 잔여 리스크

* `EnvironmentSpec`은 NodeKit 내부에서는 보존되지만, 외부 `api-protos`의 `nodeforge.proto`에 아직 필드가 없어 gRPC 전송은 완결되지 않았다.
